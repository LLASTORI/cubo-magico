# Palavras em Automações — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an "ignore keywords" field to Social Listening so comments matching automation trigger words (ManyChat etc.) are skipped by AI, excluded from stats, and hidden from the comments list by default — with a toggle to reveal them.

**Architecture:** New DB columns (`ignore_keywords` on `ai_knowledge_base`, `is_automation` on `social_comments`). At `process_ai` time, matching comments get `ai_processing_status='skipped'` + `is_automation=true`. Frontend filters `is_automation=false` by default; toggle "Ver respostas de automações" reveals them. Matching is starts-with + case-insensitive.

**Tech Stack:** PostgreSQL migration (Supabase), Deno Edge Function (TypeScript), React + TanStack Query, shadcn/ui (Switch, Badge, Label)

**Spec:** `docs/superpowers/specs/2026-03-21-ignore-keywords-design.md`

```typescript
function matchesIgnoreKeywords(text: string, keywords: string[]): boolean {
  if (!keywords.length) return false;
  const normalized = text.toLowerCase().trim();
  return keywords.some(kw => {
    const k = kw.toLowerCase().trim();
    return k.length > 0 && normalized.startsWith(k);
  });
}
```
